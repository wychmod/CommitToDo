## 0. 使用方法与还原优先级

此文档不是风格建议，而是 V3 首页的实现合同。其他模型或开发者必须按以下优先级处理冲突：

1. 基准截图中的位置、比例、文案和视觉结果优先。
2. 本文的像素坐标、尺寸、颜色和动画参数次之。
3. `commit-homepage-light-dark-design-tokens.md` 中的共享语义 token 再次之。
4. 现有 `landing-screen.tsx` 是另一版启动页，不得作为此页面的 DOM 或视觉依据。
5. 未明确出现的营销模块不得自行增加，包括客户 Logo、价格卡、FAQ、截图轮播和评价。

### 0.1 像素还原验收环境

```text
Viewport: 1576 × 1590 CSS px

Color space: sRGB

Font smoothing: browser default

```

### 0.2 坐标说明

- 坐标原点为截图左上角 `(0, 0)`。

### 2.2 色彩使用规则

1. `#80E48C` 只用于品牌强调、主按钮、主分支、完成状态和关键数字。
2. 大面积背景必须为纯黑或近黑，不使用绿色大渐变。
3. 工作台面板主要由 `#121313` 一组相邻色构成；截图里的细微变化可用
   `linear-gradient(135deg, #111312 0%, #121414 55%, #101211 100%)` 模拟。
4. 边框不高于 `#30332F`；普通分隔线使用 `#181C19`。
5. Hero 粒子允许白色、浅绿和灰色发光；普通 UI 不允许泛光。
6. 青色只表示 `launch`，蓝色只表示 `design`；不得作为随机装饰。

### 2.3 阴影和光效

| 元素 | 阴影/光效 |
| --- | --- |
| 顶栏 | 无阴影，仅底部分隔线 |

| COMMIT 终点 | 白色双环 + 32-48px 白色 bloom，是全页最强发光点 |

## 6. Hero 粒子任务流视觉

这是首页最重要的视觉资产，不能用静态渐变或普通横线代替。

### 6.1 渲染分层

从下到上：

1. 黑色背景。
2. 低密度灰白噪点 Canvas。
3. 三条主分支 SVG 曲线。
4. 沿曲线运动的白色/绿色粒子 Canvas。
5. 文档小图标、节点和标签 DOM/SVG。
6. TODO 与 COMMIT 端点光效。
7. 产品工作台，覆盖粒子视觉的下沿。

建议结构：

```html
<div class="hero-flow" aria-hidden="true">
  <canvas class="hero-flow__particles"></canvas>
  <svg class="hero-flow__paths" viewBox="0 0 1576 440">...</svg>
  <div class="hero-flow__nodes">...</div>
</div>
```

### 6.2 视觉范围

- 可见范围约 `x=250..1335 y=224..434`。
- 端点中心约：TODO `(348, 350)`，COMMIT `(1223, 350)`。
- 中央汇聚区约为 `x=735..930`。
- 工作台顶部位于 y=437，因此粒子在 y=432 前渐隐，不能穿透面板。

### 6.3 路径定义

以 1576×440 viewBox 为基准，建议使用以下三条核心路径：

```svg
<path id="flow-main"
  d="M348 350 C520 350 600 350 790 352 C960 352 1080 350 1223 350" />
<path id="flow-upper"
  d="M348 350 C520 350 565 300 650 300 C825 300 1010 292 1223 350" />
<path id="flow-lower"
  d="M348 350 C520 350 570 405 650 405 C825 405 1010 410 1223 350" />
```

实际曲线必须平滑，无锐角。主线 1.2px 白灰，分支线 1px 白灰；绿色活动段
可覆盖一条 1px `#80E48C` 的低透明路径。

### 6.4 粒子参数

| 参数 | 桌面规定值 |
| --- | --- |
| 总粒子数 | 3200-4200，建议 3800 |
| 普通粒子半径 | 0.35-1.15px |
| 高亮粒子半径 | 1.2-2.2px，不超过总数 2% |
| 白粒子颜色 | `rgb(245 245 242 / 0.08..0.82)` |
| 绿粒子颜色 | `rgb(128 228 140 / 0.15..0.9)` |
| 速度 | 18-42px/s，按粒子随机 |
| 横向抖动 | ±2px |
| 法线方向噪声 | 标准差 8-22px，汇聚点降低 |
| 尾迹 | Canvas 每帧以 8%-14% 黑色覆盖实现 |
| 帧率 | 目标 60fps；低性能降到 30fps |

粒子分布不能均匀。TODO 后 80px 密度中等，中央分支区域密度最高，COMMIT 前
120px 形成强汇聚和白色喷射。背景零散粒子使用 0.03-0.12 alpha。

### 6.5 节点与小图标

- TODO 中心为 22×22 绿色圆角方块，圆角 5px；外环约 56px，双层描边。
- TODO 标签位于中心下方约 45px，等宽 16px：`TODO`。
- 上、下活动节点分别约位于 `(790, 301)`、`(790, 406)`，直径 14px，绿色。
- 主线中央节点约 `(790, 352)`，直径 14px，白色。
- 文档图标约 8-10 个，12-14px，灰色 35%-55% 不透明度；分布于三条路径之间。
- COMMIT 中心为 24px 白色勾，外圈约 55px，外部 80-110px 白色 bloom。
- COMMIT 标签位于中心下方约 45px，等宽 16px：`COMMIT`。

## 11. 动画系统

静态截图展示的是动画结束状态。以下为 V3 首页的最终动画规定。

### 11.1 首次加载总时间轴

| 时间 | 元素 | 动作 |
| --- | --- | --- |
| 0-180ms | 页面背景 | 从纯黑立即出现，不做白屏淡入 |
| 80-420ms | 顶栏 | opacity 0→1，Y -8→0 |
| 180-700ms | Hero 品牌 | opacity 0→1，Y 14→0 |
| 300-820ms | 中文主张 | opacity 0→1，Y 12→0 |
| 420-880ms | 描述 | opacity 0→1，Y 8→0 |
| 520-940ms | CTA | opacity 0→1，Y 8→0，两个按钮间隔 70ms |
| 260-1100ms | 粒子背景 | 粒子密度 0→100%，路径逐渐可见 |
| 700-1450ms | TODO→COMMIT | 一次性高亮粒子沿三条路径通过 |
| 900-1500ms | 工作台 | opacity 0→1，Y 18→0，scale .992→1 |
| 1500ms 后 | Hero | 进入低频常驻动画 |

所有进入动画使用 `--v3-ease-enter`，不得弹簧过冲。

### 11.2 常驻 Hero 动画

- 粒子持续从 TODO 流向 COMMIT，单次旅程 5.5-8s，随机错开。
- 绿色活动节点每 2.4s 呼吸一次：scale 1→1.12→1，alpha 0.7→1→0.7。
- COMMIT 外辉光每 3.2s 轻微变化：blur 32→44→32px，透明度变化不超过 12%。
- 文档图标以 8-14s 的周期沿路径移动 10-24px，不旋转。
- 背景点阵不动画，避免整页持续闪动。
- 工作台内容不自动滚动、不轮播。

### 11.3 滚动触发

- “为什么选择”标题进入视口 75% 时淡入，Y 16→0，500ms。
- 四张卡按 70ms stagger 依次淡入，Y 18→0，520ms。
- 底部 CTA 进入视口 82% 时淡入，Y 12→0，420ms。
- 动画只执行一次；返回滚动位置不反复播放。

### 11.4 Hover / Active

| 元素 | Hover | Active |
| --- | --- | --- |
| 主按钮 | 120ms 变亮、Y -1、轻辉光 | 80ms Y 0、压暗 |
| 次按钮 | 背景变 `#141716`，边框增强 | 背景 `#101312` |
| 顶栏链接 | 图标/文字变白，弱背景出现 | 背景再压暗 |
| 卖点卡 | 边框变 `#465048`，Y -2 | Y 0 |
| 工作台可点击行 | 深色弱底出现 | 保持选中标识 |
| 热力格 | scale 1.12，显示 tooltip | scale 1.0 |

### 11.5 Reduced motion

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    scroll-behavior: auto !important;
    animation-duration: 1ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 1ms !important;
  }

  .hero-flow__particles {
    display: none;
  }

  .hero-flow__paths,
  .hero-flow__nodes {
    opacity: 1;
  }
}
```

Reduced motion 下必须保留静态分支线、节点、TODO 和 COMMIT，不能让 Hero 变空。

### 17.5 截图差异标准

- 容器边界平均误差：≤ 2px。
- 主要文本基线误差：≤ 3px。
- 大面积色块 RGB 单通道平均误差：≤ 6。
- 非粒子区域像素差异率目标：≤ 3%。
- 粒子区域使用结构掩码比较，不要求逐粒子位置相同；要求路径、密度和亮度分布一致。

## 18. 禁止事项

4. 不把四张卖点卡做成大圆角、玻璃拟态或浮空卡。
5. 不省略粒子任务流，也不以单张静态背景图永久替代。
6. 不让导航、卡片或工作台出现超过 12px 的普通圆角。

8. 不为演示工作台加入滚动条、表单编辑或真实数据加载。
9. 不在移动端压缩五列工作台导致文字重叠；应横向滚动。
10. 不使用随机 lorem ipsum 或替换本文规定文案。
