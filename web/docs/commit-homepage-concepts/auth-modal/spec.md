## 1. 背景

CommitToDo 当前首页已经具备完整 V3 品牌视觉，包含顶部导航、Hero、Git 分支粒子、
工作台预览和 local-first 信任表达。当前顶部“登录”和“开始使用”均直接跳转
`/workspace`，项目没有认证状态、认证表单或登录后的账号表达。

本功能需要按照视觉基准，在首页上增加一个认证模态框。它不是独立白底登录页，也
不接入真实后端。账号与密码固定为 `admin` / `admin`，但登录、注册、忘记密码、
GitHub 模拟登录、记住我、退出登录、错误反馈、路由恢复和响应式体验都必须形成
可运行闭环。

CommitToDo 的本地优先定位保持不变：未登录用户仍可进入本地工作台；模拟登录仅用于
展示未来账号和同步体验，不得阻止用户访问本地数据。

---

## 2. 目标

- 在不破坏现有首页视觉和动画的前提下增加高保真认证模态框。
- 让用户可以使用固定演示凭据 `admin` / `admin` 完成登录并进入工作台。
- 登录后在应用壳层看到明确的 `admin` 账号状态，并能正常退出。
- 让注册、忘记密码和 GitHub 登录具有完整、诚实的模拟流程。
- 支持刷新恢复、直接链接、键盘、移动端、加载、成功和失败状态。
- 通过组件测试和真实浏览器端到端验证证明所有流程可用。

---

## 3. 核心产品决策

### D1. 首页模态框，而不是脱离首页的独立认证画布

桌面端在首页上方显示居中模态框，保留被压暗的首页、Hero 分支流和工作台预览。
原因是首页已经承担品牌、产品能力和 local-first 信任说明；认证流程较短，跳离首页
会产生视觉断裂。

移动端认证界面表现为占满可用视口的全屏对话框，但仍属于首页认证流程。

### D2. 登录可选，不建立强制路由守卫

“开始使用”和 Hero 的“进入工作台”继续允许用户以本地模式进入 `/workspace`。
“登录”打开认证流程。登录成功后同样进入 `/workspace`，但用户菜单显示已登录状态。

这样既满足模拟认证演示，也不违背首页“无需注册即可使用”和 local-first 的承诺。

### D3. 演示认证必须透明

- 唯一可登录凭据：账号 `admin`，密码 `admin`。
- 不发送任何认证网络请求。
- 不生成、保存或解析真实 Token。
- 注册不会创建真实账号。
- 忘记密码不会发送邮件或短信。
- GitHub 按钮不会打开 OAuth，也不会访问 GitHub。
- 所有模拟流程都要通过界面文案明确标识为演示行为，不能让用户误以为数据已上传。

### D4. 图片“邮箱”字段改为“账号”

视觉基准显示“邮箱”，但固定凭据 `admin` 不是邮箱。为避免无意义的邮箱格式校验和
浏览器错误提示，字段标签改为“账号”，占位符为“输入账号”。布局、图标和尺寸仍
按照图片还原。

## 4. 设计规格

### 4.1 Purpose Statement

认证弹窗用于演示 CommitToDo 未来的账号和跨设备同步入口，同时保持本地模式随时
可用。用户应在不离开首页品牌上下文的情况下完成认证，并清楚知道当前只是本地模拟，
不会产生真实账号或云端数据。

### 4.2 Aesthetic Direction

**Industrial / utilitarian Git workbench**：工业化、克制、高密度但清晰的暗色工具界面。

视觉吸引力来自精确对齐、分支线叙事、有限的绿色强调和稳定的明暗层级。禁止把它
做成通用 SaaS 登录卡、玻璃拟态面板或营销式左右分屏。

### 4.3 Color Palette

认证弹窗必须消费现有 V3 Token，不在组件中散落裸色：

| 角色 | Token | 视觉基准 |
| --- | --- | --- |
| 页面背景 | `--v3-bg` | `#000000` |
| 弹窗表面 | `--v3-card` / `--v3-bg-near` | 接近 `#0c0f0d` |
| 输入表面 | `--v3-bg-near` | 接近纯黑 |
| 边框 | `--v3-border` | `#30332f` |
| 分隔线 | `--v3-divider` | `#181c19` |
| 强文本 | `--v3-text-strong` | `#f5f5f2` |
| 次级文本 | `--v3-text-secondary` | `#aeb4af` |
| 主行动 | `--v3-primary` | `#80e48c` |
| 分支辅助 | `--v3-launch` | `#59cbd0` |
| 错误 | `--v3-danger` | `#e6635b` |

不得修改 landing 私有样式来实现认证弹窗。认证功能应复用 V3 Token 或新增自包含的
认证样式文件。

### 4.4 Typography

项目品牌字体属于既有设计系统，本功能沿用：

- 中文和正文：`--v3-font-sans`。
- 分支、状态和演示凭据：`--v3-font-mono`。
- 弹窗主标题：28-30px，700。
- 品牌名：18px，600。
- 标签和正文：13-15px。
- 辅助与可信说明：12px。
- 字母间距保持 0；只有小型英文眉题可使用现有 V3 uppercase 规则。

### 4.5 Layout Strategy

视觉基准为 1536 x 1024：

- 顶部首页导航保持可辨认，但认证打开后不可交互。
- 全屏遮罩压暗首页，背景内容仍能辨认，不做重度模糊。
- 弹窗桌面宽度约 488px，高度随内容变化，最大高度不超过视口减 48px。
- 弹窗位于视口水平居中、略高于垂直中心。
- 弹窗圆角不超过 8px，1px 边框，使用克制面板阴影。
- 内容顺序：品牌栏 → 分支线 → 标题说明 → 登录/注册 Tab → 表单 →
  可信说明。
- 弹窗内部不再嵌套卡片。
- 小于 768px 时改为全屏对话框，表单独立滚动，主按钮始终可达。

### 4.6 图片关键视觉拆解

1. 首页整体亮度被压低，顶部导航和工作台仍作为品牌环境存在。
2. 弹窗顶部左侧为白底 Git 分支 Logo + `CommitToDo`，右侧为关闭按钮。
3. 品牌栏下是一条细小的多分支流线，使用主绿、青色和提交勾节点。
4. 主标题“欢迎回来”高于说明文字，左对齐。
5. “登录 / 注册”为平直 Tab，不做胶囊；当前项使用绿色文字和底线。
6. 输入框高约 42-48px，图标在左，密码可见性按钮在右。
7. “记住我”和“忘记密码”同一行，主按钮占满宽度。
8. GitHub 为描边次按钮；账号创建入口为低权重文本链接。
9. 底部可信说明独立于表单，使用盾牌图标和绿色状态点。

## 5. 功能需求

### F1. 打开与关闭认证流程

- 点击首页顶部“登录”打开登录 Tab。
- 直接访问 `/login` 时显示首页背景和登录 Tab。
- 直接访问 `/signup` 时显示首页背景和注册 Tab。
- 点击关闭按钮、按 `Escape` 或点击遮罩空白区域关闭弹窗并回到 `/`。
- 关闭后焦点返回触发认证的按钮。
- 弹窗打开时锁定页面背景滚动和背景交互。

### F2. 登录 Tab

登录表单包含：

- 账号输入框，标签“账号”，占位符“输入账号”。
- 密码输入框，标签“密码”，占位符“输入密码”。
- 密码显示/隐藏按钮。
- 默认选中的“记住我”复选框。
- “忘记密码”入口。
- 主按钮“登录”。
- 分隔文字“或”。
- “使用 GitHub 继续”模拟按钮。
- “还没有账户？创建账户”注册入口。

### F3. 登录校验与结果

- 空账号提示“请输入账号”。
- 空密码提示“请输入密码”。
- 任一字段不是 `admin` 时提示“账号或密码不正确，请使用 admin / admin”。
- 错误显示在表单内，使用 `aria-live`，不以浏览器 alert 表现。
- 输入改变后移除对应字段的旧错误。
- 提交时进入短暂、可观察的 loading，禁用重复提交。
- `admin` / `admin` 登录成功后显示成功反馈并进入 `/workspace`。
- 登录成功后，应用用户菜单显示账号 `admin` 和演示登录状态。

### F4. 记住我

- 勾选时，关闭并重新打开浏览器后仍保持模拟登录状态。
- 不勾选时，只在当前浏览器会话保持登录；新会话恢复本地未登录模式。
- 不得在任何存储中保存明文密码。
- 仅保存最小模拟会话信息，如账号、登录状态和记住方式。

### F5. 注册 Tab

注册表单包含：

- 账号。
- 密码。
- 确认密码。
- 服务条款与隐私政策同意复选框。
- 主按钮“创建账户”。
- “已有账户？返回登录”入口。

注册校验：

- 三个输入均必填。
- 密码与确认密码必须一致。
- 必须勾选协议。
- 密码最少 5 个字符；本演示允许输入任意符合长度的值以体验校验。

注册结果：

- 提交成功后不创建真实账号。
- 显示“演示注册完成，请使用 admin / admin 登录”。
- 自动切换回登录 Tab，并将账号预填为 `admin`；密码不自动明文填入。

### F6. 忘记密码模拟流程

- 点击“忘记密码”后，在同一个弹窗内切换为重置视图，不再打开第二层弹窗。
- 显示说明：演示环境不会发送邮件，固定密码为 `admin`。
- 提供“恢复演示账号”主按钮和“返回登录”入口。
- 点击恢复后回到登录 Tab，账号预填 `admin`，显示“演示密码已恢复为 admin”。

### F7. GitHub 模拟登录

- 点击后显示 GitHub 按钮 loading 状态。
- 不打开新窗口，不访问 GitHub，不请求 OAuth。
- 短暂模拟后以 `admin` 身份登录并进入 `/workspace`。
- 成功反馈明确写“GitHub 演示登录成功”，避免冒充真实授权。

### F8. 登录后账号状态与退出

- 已登录时，应用顶部用户菜单的头像显示 `A` 或 `admin` 的可识别首字母。
- 用户菜单显示“admin”和“演示账号”。
- 提供“退出登录”动作。
- 退出后清除模拟会话，保留所有 IndexedDB 本地业务数据和设置。
- 退出后进入 `/`；首页可再次打开认证弹窗。
- 未登录进入工作台时继续显示现有“本地模式”，不能显示错误或强制登录。

### F9. 首页入口行为

- 顶部“登录”打开登录 Tab。
- 顶部“开始使用”、Hero“进入工作台”和底部 CTA 继续直接进入本地工作台。
- 注册链接和 `/signup` 打开注册 Tab。
- 已登录用户点击首页“登录”时可直接进入工作台，或打开账号状态；最终实现必须选择
  一种稳定行为并在技术计划中固定。推荐直接进入 `/workspace`。

### F10. URL 与刷新恢复

- `/login` 与 `/signup` 可直接访问和刷新。
- 登录/注册 Tab 切换时 URL 与当前状态一致。
- 浏览器前进/后退能正确打开、切换或关闭认证弹窗。
- 未知认证子状态回退到登录 Tab，不显示空白页。

### F11. 键盘与焦点

- 打开登录时焦点进入账号输入框。
- 注册时焦点进入注册账号输入框。
- Tab 键焦点限制在弹窗内。
- `Enter` 提交当前有效表单。
- `Escape` 关闭弹窗。
- 密码可见按钮、复选框、Tab、链接、关闭按钮都可键盘操作。
- 关闭后焦点返回原触发按钮。

### F12. 响应式行为

- 1536 x 1024：接近视觉基准比例，弹窗与首页背景完整可辨。
- 1024 x 768：弹窗仍居中，内容不裁切。
- 768 x 1024：弹窗可收窄，背景导航简化。
- 390 x 844 与 360 x 800：认证改为全屏对话框，表单可滚动，无横向滚动。
- 移动键盘弹出时当前输入和主按钮可达。

### F13. 动效与状态过渡

- 遮罩使用短暂淡入；弹窗使用轻微上移和透明度进入，不缩放过度。
- Tab 切换不改变弹窗宽度，避免布局抖动。
- 登录、注册、恢复和 GitHub 模拟均有 loading、成功和错误反馈。
- `prefers-reduced-motion` 下取消位移动画，只保留即时状态变化。

## 7. 非功能需求

### N1. 视觉一致性

- 认证弹窗使用现有 V3 色彩、字体、圆角、边框、焦点环和动效语义。
- Logo、图标统一使用项目现有图标体系；不使用 Emoji 或手绘 SVG 图标替代 Lucide。
- 不改变首页 Hero、工作台预览、价值区和底部 CTA 的布局与视觉。
- 不在全局入口无条件导入认证私有样式。

### N2. 可访问性

- 对话框具有名称、描述和 modal 语义。
- 表单标签与输入建立程序化关联。
- 错误、成功和 loading 对读屏器可感知。
- 文本和控件对比度达到 WCAG AA。
- 交互目标在移动端至少 44 x 44px。

### N3. 安全与诚实性

- 不保存密码，不记录密码到日志、URL、Toast 或错误监控。
- 演示凭据不是安全边界，代码和文案都不得暗示真实安全认证。
- 不执行任何认证、注册、重置密码或 OAuth 网络请求。
- 退出登录不能清理 IndexedDB 业务数据。

### N4. 性能

- 认证弹窗不复制或重新实例化首页粒子系统。
- 打开弹窗不触发首页数据重载。
- 动画保持合成友好，不对整个首页实时应用重度滤镜。
- 登录弹窗代码允许按需加载，但不能导致首次点击长时间空白。

### N5. 工程质量

- React 组件显式返回 `JSX.Element`。
- 使用 TypeScript 严格模式，不使用 `any`。
- 状态逻辑和纯函数具有单元测试；弹窗具有行为测试。
- 代码遵循现有 Presentation/Core 分层，不引入 Data/Repository 依赖。

## 9. 不做的事

- 不接入真实后端、数据库用户表或云端同步。
- 不实现 JWT、Cookie Session、Refresh Token 或权限系统。
- 不实现真实邮箱验证码、短信验证码、邮件发送或密码重置。
- 不实现真实 GitHub OAuth。
- 不创建任意用户名可持久登录的本地账号系统。
- 不新增团队、组织、角色、订阅或计费能力。
- 不强制未登录用户离开本地工作台。
- 不重做首页或修改 landing 参考视觉。
- 不在本阶段实现代码；必须先批准后续 `plan.md`、`task.md`、`checklist.md`。

## 10. 验收标准

### AC1. 视觉还原

在 1536 x 1024 下，弹窗宽度、位置、遮罩、品牌栏、分支线、Tab、输入、主按钮、
GitHub 按钮和可信说明与视觉基准具有一致的信息结构，无重叠、裁切或明显比例偏差。

对应：F1、F2、F12、F13、N1。

### AC2. 登录成功

输入 `admin` / `admin` 并提交，显示 loading 和成功反馈，然后进入 `/workspace`；
用户菜单显示 `admin` 和演示账号状态。

对应：F2、F3、F8。

### AC3. 登录失败

空字段显示各自必填错误；错误凭据显示 admin/admin 提示；错误可被修改清除；表单
不导航、不写入登录状态。

对应：F3。

### AC4. 记住我

勾选后新浏览器会话仍显示 admin 登录态；不勾选时新会话恢复本地模式；任何存储中
都不存在明文密码。

对应：F4、N3。

### AC5. 注册闭环

注册字段、确认密码和协议校验可观察；有效提交显示演示注册成功，切回登录并预填
`admin`，不创建真实账号、不发网络请求。

对应：F5。

### AC6. 忘记密码闭环

忘记密码在同一弹窗切换视图；恢复后返回登录、预填 `admin` 并显示密码恢复提示；
不发送网络请求。

对应：F6。

### AC7. GitHub 模拟闭环

GitHub 按钮进入 loading，随后以 admin 进入工作台并显示“演示登录成功”；没有弹窗、
跳转到 GitHub 或网络请求。

对应：F7。

### AC8. 本地模式保持

未登录用户通过“开始使用”进入工作台；退出登录后 IndexedDB 中仓库、分支、任务、
提交和设置不变。

对应：F8、F9、D2。

### AC9. URL 与历史记录

`/login`、`/signup` 可直接打开和刷新；Tab 与 URL 一致；浏览器前进/后退正确切换
或关闭弹窗。

对应：F1、F10。

### AC10. 键盘和读屏

焦点进入正确字段并限制在弹窗；Enter 提交，Escape 关闭，关闭后焦点返回；错误和
状态可被读屏器感知。

对应：F11、N2。

### AC11. 响应式

1536、1024、768、390、360 宽度均无横向滚动、文本溢出和不可达按钮；手机为全屏
对话框，软键盘打开时可完成提交。

对应：F12。

### AC12. 动效和性能

打开/切换/关闭没有明显布局跳动；减少动态效果偏好被尊重；弹窗不复制粒子画布，
不触发首页数据重载。

对应：F13、N4。

### AC13. 工程门禁

认证相关测试通过，且 `npm run lint`、`npx tsc -b`、`npm test`、`npm run build`
全部通过；首页既有行为测试无回归。

对应：N5。

## 12. 审批时需要确认的产品假设

本规格已采用以下推荐假设：

1. 本地模式不需要登录，现有“开始使用”继续直接进入工作台。
2. 图片中的“邮箱”改为“账号”，以便合法输入 `admin`。
3. 注册接受任意满足前端规则的演示输入，但不会创建该凭据；完成后提示使用
   `admin` / `admin` 登录。
4. GitHub 登录直接模拟为 admin 登录，不打开 OAuth。
5. 已登录用户点击首页“登录”时推荐直接进入工作台。

批准本规格即表示接受以上 5 项假设。若其中任一项需要调整，应先修改本规格，再进入
技术设计阶段。
