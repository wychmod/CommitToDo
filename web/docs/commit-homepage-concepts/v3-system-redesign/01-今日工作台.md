# V3 今日工作台像素级实现规划

> 页面：今日工作台  
> 目标路由：`/workspace`  
> 基准截图：`../今日.png`  
> 基准画布：1536 x 1024  
> 首页状态：V3 首页已实现，本页只实现用户进入应用后的工作台。  
> 目标：用户打开应用后 5 秒内知道今天该做什么，并能进入最近仓库。

## 0. 页面职责

今日工作台回答：

- 今天要做什么？
- 哪些任务逾期？
- 最近在推进哪个仓库？
- 本周节奏如何？
- 本地数据是否安全保存？

它不是仓库详情页，也不承担复杂任务筛选。复杂筛选必须进入
`/repository/:id/tasks`。

### 4.4 任务交互

| 交互 | 行为 |
| --- | --- |
| 点击任务行 | 打开右侧任务详情面板；右栏被详情面板覆盖或主区扩展出 drawer |
| 点击 checkbox | 立即完成/恢复；显示 toast，可撤销 |
| 已完成任务 checkbox | 恢复为待办，移出已完成计数 |
| 逾期任务 checkbox | 完成时仍生成“完成并提交”提醒 |
| 点击状态 chip | 打开状态菜单：待办、进行中、已完成、已取消 |
| 点击优先级 | 打开优先级菜单：高、中、低 |
| 点击仓库标题 | 进入 `/repository/:id` |
| 点击分支名 | 进入 `/repository/:id/tasks?branch=:branchId` |

快速完成不等于提交。若产品规则要求完成即提交，应在完成后弹出轻量确认：

```text
已完成任务。是否生成提交？
[稍后] [完成并提交]
```

## 8. 响应式

### 8.1 1280px 以上

完全按截图三栏布局。

### 8.2 1024-1279px

- 左侧导航保持 220px。
- 右栏收为可折叠抽屉，默认展示“今日节奏”小卡。
- 主任务区占剩余宽度。

### 8.3 768-1023px

- 左侧导航图标化为 72px。
- 顶栏搜索缩短。
- 右栏移动到主任务列表下方。

### 8.4 小于 768px

- 顶栏简化为 Logo、搜索、新建、用户。
- 底部导航：今日、仓库、洞察、搜索。
- 今日任务第一屏优先展示。
- 右栏卡片依次排在任务列表下。
- 任务行元信息换行，状态 chip 不挤压任务标题。

## 9. 可访问性

- 任务列表使用 `role="list"`，任务行用 button 或可聚焦 article。
- checkbox 使用真实 `input[type=checkbox]` 或等价 ARIA。
- 环形进度提供文本：`今日完成 3 个，共 6 个，完成率 50%`。
- 热力图格子有 `aria-label="7 月 2 日，完成 2 个任务"`。
- `Ctrl+K` 搜索入口必须可键盘聚焦。
- 所有图标按钮有 `aria-label`。

## 10. 数据需求

页面需要聚合：

- 所有仓库。
- 所有分支。
- 今日任务、逾期任务、待提交任务。
- 本周完成记录。
- 最近仓库，按 `updatedAt` 倒序。
- 本地存储状态。

建议 ViewModel：

```ts
interface TodayWorkspaceViewModel {
  dateLabel: string;
  progress: { done: number; total: number; overdue: number; estimateMinutes: number };
  taskGroups: TodayTaskGroup[];
  weekHeatmap: HeatmapDay[];
  recentRepositories: RecentRepository[];
  storage: { status: 'saved' | 'saving' | 'offline' | 'error'; label: string };
}
```
