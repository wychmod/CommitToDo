# V3 仓库概览像素级实现规划

> 页面：仓库概览  
> 目标路由：`/repository/:id`  
> 基准截图：`../仓库概览.png`  
> 基准画布：1672 x 941  
> 目标：用户进入仓库后快速理解分支、重点任务、最近提交和长期节奏。

## 0. 页面职责

仓库概览回答：

- 这个仓库当前整体进展如何？
- 有哪些分支，哪个是当前分支？
- 每个分支有哪些重点任务？
- 最近发生了哪些提交？
- 过去 12 个月节奏如何？

它不承担完整任务管理、复杂筛选和批量操作；这些必须进入仓库任务页。

## 3. 标题区

标题区从 `x=291 y=95` 开始。

内容必须匹配截图：

```text
LOCAL REPOSITORY
运动计划
把 5km 训练拆成可以持续推进的分支
● 最近活动 · 12 分钟前
```

右侧按钮：

```text
+ 新建分支
+ 新建任务
```

规格：

- 英文眉题 `LOCAL REPOSITORY`：12px，绿色，等宽或半等宽，字距 0.08em。
- 标题 `运动计划`：48-54px，白色，字重 700。
- 描述：17px，`#AEB4AF`。
- 最近活动：13px muted，左侧绿色圆点 8px。
- `新建分支` 次按钮：124 x 44，透明底、1px 边框。
- `新建任务` 主按钮：124 x 44，绿色底、近黑字。
- 标题区底部无独立卡片背景，与页面画布融合。

行为：

- 新建分支：打开分支弹窗，默认基于当前分支创建。
- 新建任务：打开任务表单面板，默认仓库为当前仓库，分支为当前分支。

## 8. 分支和按钮关系

仓库概览中每个按钮必须有明确目标：

| 元素 | 行为 |
| --- | --- |
| 顶栏仓库选择器 | 切换仓库后进入新仓库 `/repository/:id` |
| `+ 新建分支` | 打开分支创建弹窗 |
| `+ 新建任务` | 打开任务表单面板 |
| 分支卡 | 切换当前分支上下文 |
| 任务行 | 打开任务详情面板 |
| 任务 checkbox | 完成/恢复任务 |
| `查看全部分支` | 打开分支管理弹窗 |
| `查看全部任务` | 进入仓库任务页 |
| `查看全部提交` | 进入洞察活动标签 |
| `查看完整洞察` | 进入洞察节奏标签 |

## 9. 弹窗与面板

### 9.1 新建分支弹窗

宽 420px，居中。

字段：

- 分支名称。
- 基于分支，默认当前分支。
- 颜色：绿、青、蓝、紫。
- 描述，可选。

保存后：

- 创建分支。
- 弹窗关闭。
- 分支列表新增卡片。
- 当前分支切换为新分支。

### 9.2 任务表单面板

从右侧滑出，宽 420-460px。字段与今日页一致，但仓库不可更改或默认当前仓库。

### 9.3 任务详情面板

右侧覆盖当前右栏或从右侧浮出，宽 420px。

包含：

- 任务标题。
- 状态/优先级。
- 所属仓库和分支。
- 截止日期。
- 关联提交。
- 操作：编辑、完成并提交、删除。

## 10. 空状态

### 10.1 仓库没有分支

显示：

```text
还没有分支
创建 main 分支后开始拆分任务。
[新建分支]
```

### 10.2 分支没有任务

当前推进组显示：

```text
main 暂无任务
给这个分支添加第一条任务。
[新建任务]
```

### 10.3 没有提交记录

最近提交面板显示：

```text
还没有提交
完成任务后会在这里留下记录。
```

### 10.4 没有热力数据

热力图仍显示空格矩阵，统计为 0，不隐藏面板。

## 11. 响应式

### 11.1 1280px 以上

按截图布局：左侧导航、主双列、右侧信息栏完整显示。

### 11.2 1024-1279px

- 右侧信息栏移动到主内容下方，形成两张横向卡。
- 主内容区仍保留分支概况 + 当前推进两列。

### 11.3 768-1023px

- 左侧导航图标化或抽屉。
- 分支概况和当前推进改为上下排列。
- 最近提交和热力图上下排列。

### 11.4 小于 768px

- 顶部显示仓库名、当前分支、更多菜单。
- 分支列表通过底部 sheet 打开。
- 当前推进优先显示。
- 提交和热力图在任务摘要之后。

## 12. 可访问性

- 分支卡使用 button，并通过 `aria-pressed` 标识当前分支。
- 任务摘要列表支持键盘进入任务详情。
- 提交时间线节点配文本，不只用颜色。
- 热力图格子配日期和数量 `aria-label`。
- 新建分支和新建任务按钮有明确文本。
