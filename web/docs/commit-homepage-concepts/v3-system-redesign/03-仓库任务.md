# V3 仓库任务像素级实现规划

> 页面：仓库任务  
> 目标路由：`/repository/:id/tasks`  
> 基准截图：`../仓库任务.png`  
> 基准画布：1672 x 941  
> 目标：用户在当前仓库和分支中创建、筛选、更新、完成和提交任务。

## 0. 页面职责

仓库任务回答：

- 当前分支下一步做什么？
- 哪些任务待办、进行中、已完成或已取消？
- 如何按状态、优先级、截止日期筛选？
- 如何查看任务详情和关联提交？
- 如何完成任务并形成提交记录？

它是最主要的任务操作页面，信息密度最高。

## 8. 新建/编辑任务表单面板

桌面端右侧 panel，宽 420px，覆盖任务详情面板。

字段顺序：

1. 标题，必填。
2. 描述。
3. 所属分支，默认当前分支。
4. 状态。
5. 优先级。
6. 截止日期。
7. 提醒时间。

按钮：

- 新建态：`取消`、`创建任务`。
- 编辑态：`取消`、`保存更改`。
- 完成态可出现 `完成并提交`。

表单验证：

- 标题不能为空。
- 标题最大长度建议 100。
- 截止日期不能是无效日期。
- 仓库/分支必须存在。

## 9. 完成并提交流程

用户点击 `完成并提交` 后打开确认面板或弹窗。

内容：

```text
完成并提交
将任务标记为已完成，并生成一条提交记录。

提交类型 [feat v]
提交标题 [完成晨跑 5 km]
说明     [可选]

[取消] [完成并提交]
```

成功后：

- 任务状态变为已完成。
- 生成 Commit。
- 详情面板显示相关提交。
- 今日工作台进度更新。
- 洞察活动和图谱更新。

失败后：

- 任务状态不改变或事务回滚。
- 显示错误 toast，提供重试。

## 10. 分支管理

`管理分支` 打开 dropdown 或 modal。

功能：

- 新建分支。
- 重命名当前分支。
- 合并到 main。
- 归档分支。
- 删除分支。

危险操作必须二次确认。

合并分支时：

- 生成 merge commit。
- 将分支任务迁移或标记已合并，按产品规则决定。
- 洞察图谱出现合并边。

## 11. 筛选、搜索和排序规则

筛选优先级：

1. 当前仓库。
2. 当前分支。
3. 搜索关键词。
4. 状态。
5. 优先级。
6. 截止日期。
7. 保存视图。
8. 排序。

搜索匹配：

- 任务标题。
- 描述。
- 分支名。
- 关联提交标题/哈希。

排序：

- 按更新时间：最近更新在前。
- 按截止日期：逾期、今天、未来、无日期。
- 按优先级：高、中、低。
- 按创建时间：最近创建在前。

## 13. 响应式

### 13.1 1280px 以上

按截图完整显示筛选栏、表格、详情面板。

### 13.2 1024-1279px

- 详情面板改为抽屉，默认收起。
- 表格占用右侧空间。
- 筛选栏保持。

### 13.3 768-1023px

- 左侧全局导航图标化。
- 筛选栏收为顶部筛选按钮。
- 详情为右侧 drawer。

### 13.4 小于 768px

- 分支 tab 横向滚动。
- 任务表格改为列表卡片，但不能嵌套卡片。
- 任务详情为全屏页面。
- 新建/编辑任务为全屏表单或底部 sheet。
- 返回列表保留分支、筛选和滚动位置。

## 14. 可访问性

- 表格使用语义 `table` 或 ARIA grid。
- 行 selection 使用 `aria-selected`。
- checkbox 可键盘操作。
- 筛选项使用 radio/checkbox 语义。
- 详情面板关闭按钮有 `aria-label="关闭任务详情"`。
- 删除按钮有明确 `aria-label="删除任务"`。
- Chip 不能只靠颜色表达状态，要有文字。
