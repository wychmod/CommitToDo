# V3 设置像素级实现规划

> 页面：设置  
> 目标路由：`/settings`  
> 基准截图：`../设置.png`  
> 基准画布：1536 x 1024  
> 目标：统一管理外观、提醒、本地数据、当前仓库和账户/PWA 状态。

## 0. 页面职责

设置回答：

- 如何切换外观、强调色和界面密度？
- 如何开启截止提醒和通知权限？
- 本地数据保存是否安全，如何导入导出？
- 当前仓库如何改名、改描述、改默认分支、归档或删除？
- PWA 是否可安装，账户/登录入口在哪里？

全局设置和仓库设置必须合并在一个页面，通过顶部范围切换：

```text
[应用] [工作区] [当前仓库]
```

不再保留单独的仓库设置页面。

## 3. 页面标题和范围切换

标题内容：

```text
SETTINGS
设置
管理外观、提醒、本地数据和仓库。
```

范围切换：

```text
[应用] [工作区] [当前仓库]
```

规格：

- `SETTINGS` 12px 绿色。
- `设置` 42-48px，白色，700。
- 描述 15px muted。
- segmented control 高 34px，宽约 305px。
- 当前 `应用` 为绿色主色底。
- 右侧保存提示：`✓ 所有更改已保存`。

范围规则：

- `应用`：外观、提醒、数据与备份、应用与账户。
- `工作区`：工作区名称、默认仓库、导入导出范围，未来预留。
- `当前仓库`：仓库名称、描述、默认分支、归档/删除。
- 截图默认展示应用范围，但中央仍露出当前仓库分区；实现可以保持单页全部分区，
  顶部范围只影响高亮和可编辑项。

## 4. 外观分区

顶部：`y≈209`。

标题：`外观`，20px/650。

### 4.1 主题模式

行：

```text
主题模式     [深色] [浅色] [跟随系统]
```

规格：

- label 宽约 118px。
- segmented control 高 34px，宽 289px。
- 当前 `深色` 选中：弱绿底 + 绿边。
- 未选中项透明底，边框分隔。

行为：

- 深色：`data-theme="dark"`。
- 浅色：`data-theme="light"`。
- 跟随系统：监听 `prefers-color-scheme`。
- 切换后立即预览并自动保存。

### 4.2 强调色

行：

```text
强调色      绿 青 蓝 黄 红
```

规格：

- 色点直径 32px。
- 当前绿色外侧 2px focus ring。
- 色点之间间距 18px。

推荐色值：

| 名称 | 值 |
| --- | --- |
| 绿 | `#92D970` |
| 青 | `#4CCAD4` |
| 蓝 | `#718CFF` |
| 黄 | `#F2C14E` |
| 红 | `#F0645A` |

行为：

- 点击更新 `themeColor`。
- 所有主按钮、选中态、焦点色即时更新。

### 4.3 界面密度

行：

```text
界面密度     [舒适] [紧凑]
```

行为：

- 舒适：列表行高 52-56px，面板 padding 20-24px。
- 紧凑：列表行高 42-46px，面板 padding 14-18px。
- 截图中选中 `紧凑`。

### 4.4 界面预览

预览框位置约 `x=408 y=389 w=608 h=54`。

内容：

```text
[checked] 复选框选中状态     [主按钮]      main 分支
                                      launch 分支
                                      design 分支
```

规格：

- 高 54px。
- 边框 1px，圆角 6px。
- 左侧 checkbox 24px。
- 主按钮 104 x 36。
- 分支预览为三条小线 + 节点。

预览必须实时反映主题、强调色和密度。

## 5. 提醒分区

顶部：`y≈469`。

标题：`提醒`。

行：

```text
截止提醒                         [switch on]
默认提前时间                     [1 小时 v]
通知权限                         已允许
```

规格：

- 每行高 34-44px。
- switch 宽 40px，高 22px，开启为绿色轨道。
- select 宽 136px，高 34px。
- `已允许` 使用绿色文字。

行为：

- 截止提醒关闭后，默认提前时间 disabled。
- 默认提前时间选项：15 分钟、30 分钟、1 小时、1 天。
- 通知权限若未授权，显示 `未授权` 和 `请求权限` 按钮。
- 若浏览器不支持通知，显示 `不支持`。

## 6. 数据与备份分区

顶部：`y≈617`。

标题：`数据与备份`。

### 6.1 本地数据状态

行：

```text
本地数据状态      IndexedDB · 2.4 MB · 刚刚保存
```

状态文本绿色。保存中为黄色，失败为红色并显示重试。

### 6.2 数据操作

行：

```text
数据操作   [导出 JSON] [导出 CSV] [导出 Markdown] [导入数据]
```

按钮规格：

- 高 36px。
- 图标 16px，文字 14px。
- 透明底，1px 边框。
- 间距 12px。

行为：

- 导出 JSON：导出所有仓库、分支、任务、提交、设置。
- 导出 CSV：导出任务和提交两个 CSV 或 zip。
- 导出 Markdown：按仓库生成 Markdown 摘要。
- 导入数据：打开文件选择，支持 JSON；导入前显示预览和冲突策略。

导入冲突策略：

- 合并，保留现有数据。
- 覆盖，先备份再导入。
- 取消。

### 6.3 自动备份

行：

```text
自动备份     仅保存在当前设备                   [switch off]
```

当前功能若只是本地备份，不展示云同步承诺。若未实现自动备份，switch 可 disabled，
说明文字为 `即将支持`。

## 7. 当前仓库分区

顶部：`y≈770`。

标题：`当前仓库`。

字段：

```text

仓库描述       [把 5km 训练拆成可持续推进的分支]
默认分支       [main v]
```

规格：

- label 宽 196px 或在行左侧。
- input 高 32-36px。
- input 背景 `#090D0B`，边框 `#29302D`。
- 右侧宽度约 694px。

行为：

- 仓库名称必填，失焦自动保存。
- 仓库描述可为空。
- 默认分支 select 只能选择未归档分支。

### 7.1 危险操作

底部：

```text
[归档仓库]                                      [删除仓库]
```

归档仓库：

- 次按钮，左侧 archive 图标。
- 弹窗确认：`归档后默认列表将隐藏此仓库，可在设置中恢复。`

删除仓库：

- 红色文字 + 红色图标。
- 必须强确认，输入仓库名 `运动计划` 才能删除。
- 删除包含仓库、分支、任务、提交。
- 删除后跳转 `/workspace`。

## 9. 保存反馈

设置页所有可编辑项自动保存。

状态文案：

| 状态 | 文案 |
| --- | --- |
| saved | `✓ 所有更改已保存` |
| saving | `正在保存…` |
| error | `保存失败，点击重试` |
| offline | `离线可用，本地更改已保存` |

交互：

- 修改控件后立即进入 saving。
- 保存成功 500ms 后显示 saved。
- 保存失败时控件保留用户输入，并显示行级错误。

| 安装 CommitToDo | 调用 PWA 安装 |
| 登录 | 打开登录流程 |

## 11. 导入数据流程

1. 用户点击导入数据。
2. 打开文件选择，仅接受 `.json`。
3. 解析并校验 schema。
4. 展示预览：
   - 仓库数量。
   - 分支数量。
   - 任务数量。
   - 提交数量。
5. 用户选择合并或覆盖。
6. 执行导入。
7. 成功 toast，并刷新所有 store。

错误：

- 文件格式错误。
- schema 版本不兼容。
- 浏览器存储空间不足。
- 用户取消。

## 13. 响应式

### 13.1 1280px 以上

按截图显示左侧导航、中央设置区、右侧说明栏。

### 13.2 1024-1279px

- 右侧说明栏移到中央内容下方。
- 中央表单占满剩余宽度。

### 13.3 768-1023px

- 左侧导航图标化。
- 分区锚点移动到标题下方横向滚动。
- 范围 segmented 换行到标题下。

### 13.4 小于 768px

- 单列。
- 右栏说明移到页面末尾。
- 数据操作按钮 2 列或单列。
- 危险操作上下排列。
- 顶部范围切换 sticky。

## 14. 可访问性

- 表单控件必须有可见 label。
- 色点按钮有 `aria-label="选择绿色强调色"`。
- 当前色点使用 `aria-pressed`。
- switch 使用真实 checkbox 或 `role="switch"`。
- 保存状态使用 live region。
- 删除仓库强确认 modal 聚焦管理正确。
- PWA 安装按钮状态可被屏幕阅读器读取。

## 15. 数据需求

```ts
interface SettingsViewModel {
  scope: 'app' | 'workspace' | 'repository';
  appearance: {
    themeMode: 'dark' | 'light' | 'system';
    accentColor: string;
    density: 'comfortable' | 'compact';
  };
  reminders: {
    enabled: boolean;
    defaultLeadMinutes: number;
    notificationPermission: 'granted' | 'denied' | 'default' | 'unsupported';
  };
  storage: {
    engine: 'IndexedDB';
    sizeLabel: string;
    lastSavedLabel: string;
    status: 'saved' | 'saving' | 'error' | 'unavailable';
  };
  currentRepository?: {
    id: string;
    name: string;
    description: string;
    defaultBranchId: string;
    branches: Array<{ id: string; name: string }>;
    isArchived: boolean;
  };
  pwa: {
    status: 'installable' | 'installed' | 'unsupported';
  };
  account: {
    mode: 'local' | 'signedIn';
    userLabel?: string;
  };
}
```
