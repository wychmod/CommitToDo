# V3 洞察像素级实现规划

> 页面：洞察  
> 目标路由：`/insights`  
> 基准截图：`../洞察.png`  
> 基准画布：1536 x 1024  
> 目标：统一提交记录、分支图谱和热力节奏，帮助用户回顾完成了什么。

## 0. 页面职责

洞察回答：

- 最近完成了什么？
- 提交对应哪些任务？
- 分支如何创建、推进和合并？
- 过去一段时间的行动节奏如何？

原有页面合并关系：

| 原页面 | 新位置 |
| --- | --- |
| `CommitsScreen` | 洞察 -> 活动 |
| `GitGraphScreen` | 洞察 -> 图谱，范围为全部仓库 |
| `RepoGraphScreen` | 洞察 -> 图谱，范围为指定仓库 |
| `HeatmapScreen` | 洞察 -> 节奏，范围为全部仓库 |
| `RepoHeatmapScreen` | 洞察 -> 节奏，范围为指定仓库 |

## 3. 页面标题和范围筛选

标题区内容：

```text
INSIGHTS
洞察
回顾提交、理解分支演进，并看见长期节奏。
```

规格：

- `INSIGHTS` 12px 绿色，字距 0.08em。
- `洞察` 52px，白色，700。
- 描述 16px，`#AEB4AF`。

右侧筛选：

```text
[全部仓库] [指定仓库] [过去 90 天 v] [全部分支 v]
```

位置约 `x=769 y=139`。

控件规则：

- 前两个为 segmented control，高 40px。
- 当前 `全部仓库` 为绿色主色底，近黑字。
- 时间范围 select 宽 132px。
- 分支 select 宽 116px。
- 指定仓库时显示仓库选择器；全部仓库时分支 select 只显示“全部分支”。

URL 参数：

```text
/insights?scope=all&tab=graph&range=90d
/insights?repository=:id&tab=activity
/insights?repository=:id&branch=:branchId&tab=heatmap
```

## 4. 标签系统

标签：

```text
活动   图谱   节奏
```

规格：

- 标签行顶部 `y≈239`。
- 每个标签宽 70-85px，高 38px。
- 选中 `图谱`：文字绿色 + 底部 3px 绿色线。
- 底部横线贯穿主区，`#1B211F`。

行为：

- `活动`：提交时间线列表。
- `图谱`：分支/提交图谱。
- `节奏`：热力图和统计。
- 切换标签不丢失范围筛选。

## 5. 图谱标签实现

截图展示的是图谱标签。

### 5.1 图谱画布

位置：`x=265 y=296 w=903 h=645`。

面板：

- 背景 `#07100D`。
- 边框 1px `#29302D`。
- 圆角 8px。
- 内部有时间轴网格。

标题：

```text
时间
LIFE GRAPH
主干 28 次提交 · 4 条阶段分支 · 连续推进 12 天
```

左侧时间标签：

```text
今天
7 月 7 日
7 月 5 日
7 月 3 日
```

图谱内容：

- 主干为垂直绿色线，位于画布中间偏右 `x≈758`。
- main 主干节点为绿色圆点，当前节点为双环绿色。
- launch/健康计划分支使用青色曲线。
- design/CommitToDo 发布分支使用蓝色曲线。
- 合并线使用灰白曲线。
- 时间网格为低对比虚线。

必须显示的节点/标签：

```text
人生主线 · main
merge: 发布阶段 → 人生主线
健康计划
晨跑 5 km · 建立训练节奏
学习路线
阅读 · 记录 · 形成方法
生活整理
空间清理 · 周复盘
CommitToDo 发布
视觉方向 · 交互 · 设计规范
起点 · 建立长期目标
```

### 5.2 当前节点

当前节点是 `merge: 发布阶段 → 人生主线`。

视觉：

- 中心圆点绿色。
- 外侧双环，外环直径约 38px。
- 连接标签位于右侧，白色标题，哈希和时间 muted。
- 周围轻微绿色 glow，不超过 24px blur。

点击节点：

- 右侧详情栏显示该提交。
- URL 增加 `commit=4de5f6a`。

### 5.3 缩放控件

位置：画布左下，`x≈284 y≈780 w=43 h=143`。

内容：

```text
+
100%
-
```

规格：

- 容器宽 42px，圆角 8px，边框 1px。
- 三段等高。
- 点击 `+` 放大 10%。
- 点击 `-` 缩小 10%。
- 支持鼠标滚轮、触控板缩放、拖拽平移。
- 双击画布重置或聚焦当前节点。

### 5.4 图谱算法要求

图谱必须使用稳定布局，不能每次刷新随机变化。

布局规则：

- y 轴按提交时间排序，最近在上。
- main 分支靠中间。
- 分支从 main 向左或右弯出，再合并回 main。
- 同一分支颜色稳定。
- merge commit 显示双入边或合并曲线。
- 节点点击区域不小于 24 x 24px。

## 7. 活动标签实现

活动标签替代原提交记录页面。

布局：

- 主区仍为左大列表 + 右详情栏。
- 列表按日期分组。

日期组示例：

```text
今天
10:42  feat: 完成晨跑统计卡片        运动计划 / main
16:50  merge: 发布阶段 → 人生主线     CommitToDo / launch -> main

昨天
16:50  docs: 更新需求文档             CommitToDo / main
11:33  fix: 修复登录态过期问题        CommitToDo / launch
```

行规格：

- 左侧时间 52px，等宽。
- 节点 + 竖线形成时间线。
- 提交标题 16px。
- 哈希、仓库、分支、任务数为 12-13px。
- 点击行更新右侧提交详情。

筛选：

- 复用顶部范围筛选。
- 可在列表顶部增加 `提交类型` select。

## 8. 节奏标签实现

节奏标签替代全局/仓库热力图页面。

布局：

- 顶部统计卡一行：提交数、完成任务、连续完成、单日最高。
- 中部大热力图，横向按月份。
- 底部趋势/分支贡献摘要。

内容：

```text
过去 12 个月
完成任务 45
提交 42
连续完成 12 天
单日最高 6
```

热力图：

- 桌面主区可显示 12 个月。
- 小格 13-15px，间距 4px。
- hover 显示 tooltip：日期、任务数、提交数、仓库/分支。
- 点击某天切换活动列表到该日期。

范围：

- `过去 90 天`、`过去 12 个月`、`今年`、自定义。

### 10.3 图谱过大

- 缩放。
- 拖动。
- 重置视图。
- 搜索节点。
- 当前选中节点居中。

## 11. 响应式

### 11.1 1280px 以上

按截图显示左侧导航、主图谱、右侧详情栏。

### 11.2 1024-1279px

- 右侧详情栏转为抽屉。
- 主图谱占剩余宽度。

### 11.3 768-1023px

- 左侧导航图标化或 drawer。
- 顶部筛选换行。
- 图谱区域保持最小宽度，可横向滚动。

### 11.4 小于 768px

- 顶部固定范围选择和三标签。
- 活动为纵向时间线。
- 图谱全屏可拖拽，详情从底部 sheet 打开。
- 热力图允许横向滚动，不能压缩到看不清。

## 12. 可访问性

- 标签使用 tablist/tab/tabpanel。
- 图谱提供文本摘要：提交数、分支数、当前节点。
- 节点可键盘聚焦，方向键在邻近节点间移动。
- 缩放按钮有 `aria-label`。
- 热力图格子有日期和数量说明。
- 复制哈希按钮成功后通过 live region 宣告。

## 13. 数据需求

```ts
interface InsightsViewModel {
  scope: {
    repositoryId?: string;
    branchId?: string;
    range: '90d' | '12m' | 'year' | 'custom';
    tab: 'activity' | 'graph' | 'heatmap';
  };
  commits: CommitActivity[];
  graph: {
    nodes: GraphCommitNode[];
    edges: GraphCommitEdge[];
    branches: BranchLegend[];
  };
  heatmap: HeatmapDay[];
  selectedCommit?: CommitDetail;
  stats: {
    commits: number;
    completedTasks: number;
    streakDays: number;
    bestDay: number;
  };
}
```
