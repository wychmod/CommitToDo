## Commit 项目生成后审计与优化

根据 `CLAUDE.md`、`docs/PRD.md`、`docs/system_design.md`、`docs/UI_DESIGN.md` 对项目进行了实现一致性排查，并结合 Flutter 与 UI/UX 规范做了首轮优化。

### 已优化
- 接入 `settingsProvider` 控制 `MaterialApp.themeMode`，让设置页“深色模式”开关实际生效。
- 新增 `buildLightTheme()`，并修正深色主题按钮/onPrimary/onError 的文字对比度。
- 移除 `TextScaler.noScaling`，避免破坏系统字体缩放和无障碍体验。
- 仓库删除改为事务化级联软删除关联分支与任务；分支删除改为级联软删除其任务。
- 热力图统计查询增加 `status == done` 与 `completedAt IS NOT NULL` 约束。
- 仓库详情页标题从 repositoryId 改为仓库名称，并补齐列表长按弹层中的任务删除逻辑。
- 为 RepositoryCard、TaskCard、BranchIndicator、BottomNavWidget、AppCard 加入 Semantics/InkWell，改善点击反馈和读屏语义。

### 剩余风险
- 当前环境缺少 `flutter`/`dart` 命令，无法执行 `flutter pub get`、`flutter analyze`、`flutter test`。
- drift 生成文件仍为占位内容，需在 Flutter/Dart 环境执行 `dart run build_runner build --delete-conflicting-outputs`。
- UI 仍大量使用 Material Icons 占位，后续需迁移到 Heroicons SVG；test 目录目前无测试文件。
